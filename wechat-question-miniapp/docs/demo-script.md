# Demo Script

这份脚本是给**现场演示 / 发给别人试跑**准备的，目标是让 3~5 分钟内就能把项目讲清楚。

## Demo 目标

你要传达的不是“我写了几个页面”，而是：

1. 这是一个题库搜索小程序 Demo
2. 它不只有用户侧搜索，还有后台治理与批量导入
3. 它已经具备继续往真实业务系统演进的结构

---

## 路线 1：产品视角（推荐）

### 第 1 步：首页

讲法：

- 这是一个搜题 + 题库治理 Demo
- 首页先把产品定位、推荐热词和演示顺序说清楚
- 可以直接从热词进入搜索

建议停留：20~30 秒

### 第 2 步：搜索页

重点展示：

- 输入关键词 / 热词触发搜索
- 图片搜题入口占位（说明后续可接 OCR）
- 搜索结果高亮
- 学科 / 难度 / 题型筛选
- 分组视图与分页浏览

讲法：

- 这页不是简单返回列表，而是更像真正用户会用到的搜题界面
- 如果结果很多，可以通过 facet 筛选继续缩小范围

建议停留：50~80 秒

### 第 3 步：详情页

重点展示：

- 答案摘要与完整答案切换
- 解析、标签、来源、时间等元信息
- 相关推荐

讲法：

- 搜索结果页只给“够用的信息”，详情页再承接完整内容

建议停留：20~40 秒

### 第 4 步：后台首页

重点展示：

- 管理员权限校验
- 状态统计卡片
- 模块入口

讲法：

- 这里把内容治理能力补上，不只是一个面向用户的搜题壳子

建议停留：20~30 秒

### 第 5 步：题目列表 / 编辑页

重点展示：

- 生命周期 / 审核态 / 版本号
- 归档与恢复
- 编辑页里的负责人、审核人、变更原因、来源引用
- 版本快照 / 状态流转

讲法：

- 这部分体现它更接近真实内容运营后台，而不只是 CRUD 表单

建议停留：40~70 秒

### 第 6 步：导入页

重点展示：

- 支持 JSON / JSONL / CSV / workbook manifest
- 本地暂存预览
- 云端预检
- 去重策略
- 导入结果反馈

讲法：

- 导入是这个项目比较能拉开差距的部分，因为它不是“直接写库”，而是更像正式系统的任务化流程

建议停留：40~70 秒

## 路线 2：技术视角

适合给开发同学看。

### 先讲目录

```text
miniprogram/pages/*     页面
miniprogram/utils/*     工具与 API 封装
cloudfunctions/*        云函数
data/*                  示例数据与导入模板
docs/*                  说明文档
```

### 再讲关键设计

- `question.js` 封装云函数调用
- 搜索 / 详情支持 mock fallback，便于先调 UI
- 管理动作与导入动作要求真实云函数
- 导入先做暂存与预检，再真正写入

### 最后讲可扩展点

- OCR 接入
- 真正索引分页
- 文件上传解析
- 导入任务中心
- 审核流 / 操作日志

## 演示时容易被问到的问题

### Q1：现在能不能直接商用？

更准确的回答：

- **还不是完整商用系统**
- 但它已经是一个不错的**预商用 Demo / 原型**
- 结构已经考虑了搜索、治理、导入、状态流转这些核心点

### Q2：图片搜题是真实的吗？

- 当前是演示入口 / 占位
- 已预留 OCR 文本回填的产品交互
- 后续可接真实识图能力

### Q3：导入为什么做这么复杂？

- 因为真实题库系统里，导入经常是核心链路
- 直接粘 JSON 写库太像玩具 Demo
- 加入暂存、预检、映射、去重后，更像真实业务系统
