# 题库小程序 v4 场景方案说明

这份文档聚焦“接近商业化前”的真实场景，而不是只做一个能跑的 demo。

## 1. 用户侧场景

### 场景 A：用户上传/拍摄题目图片后，希望直接搜到题

**问题**
- 用户不一定会手打完整题干
- 拍照后通常只能拿到 OCR 文本片段，存在错字、断句、漏字

**v4 方案**
- 前端把 OCR 文本统一回填到 `keyword`
- 搜索索引补充 `imageText`、`titleVariants`
- 页面文案明确提示“可粘贴图片识别出的题干片段”
- Demo 中加入“演示图片搜题入口”，便于讲解未来链路

**后续商业化增强**
- 接入真实 OCR 服务
- 做题干向量召回 / 近似匹配，而不是纯 includes
- 支持裁图、二次识别、图片纠错

---

### 场景 B：用户搜到太多结果，不知道怎么缩小范围

**问题**
- 关键词很宽泛时，结果会很多
- 只有列表没有结构，用户容易放弃

**v4 方案**
- 搜索返回 `facets`（学科、分类、难度、题型、审核态等）
- 前端支持按学科/难度/题型筛选
- 支持按学科或分类分组
- 搜索接口显式返回 `pagination`
- 页面支持上一页 / 下一页演示

**后续商业化增强**
- 记录筛选点击率，优化默认推荐顺序
- 根据用户画像优先展示常用学科/题型
- 改为数据库索引分页，而不是内存过滤后分页

---

### 场景 C：没有搜索结果

**问题**
- 用户容易把“没搜到”理解成“系统没题”
- 如果没有兜底动作，转化会直接中断

**v4 方案**
- 空结果时展示改搜建议
- 返回 `suggestions` 作为相关词兜底
- 允许从热门词、历史词、相关词继续搜

**后续商业化增强**
- 提供“反馈缺题”入口
- 把无结果词沉淀到运营后台，反向指导补库

---

### 场景 D：答案太长，用户在结果页被信息淹没

**问题**
- 用户在列表阶段只想快速判断是不是这道题
- 直接展示完整答案/长解析会造成阅读负担

**v4 方案**
- 列表只展示 `answerSummary`
- 搜索结果增加 `excerpt`
- 点击“展开更多”后再看更多答案/解析
- 详情页进一步支持完整答案展开

**后续商业化增强**
- 支持 AI 自动摘要答案
- 支持“只看结论 / 只看步骤 / 只看知识点”三层展示

---

## 2. 管理员侧场景

### 场景 E：管理员导入多个文档类型，列名不一致

**问题**
- 实际导入源可能来自 Word 转表格、Excel、CSV、旧系统 JSON、日志导出 JSONL
- 同一字段可能叫“题目 / questionTitle / title / 试题名称”

**v4 方案**
- 导入页内置多模板暂存入口：JSON / JSONL / CSV 文本
- 云函数内置字段别名表
- 允许传入 `fieldMappings`
- 统一归一到标准字段模型

**后续商业化增强**
- 提供真正的上传解析能力，而不只是文本暂存
- 支持保存机构级模板映射规则

---

### 场景 F：导入前需要 staging，而不是直接写库

**问题**
- 商业场景里，直接写库风险很大
- 管理员更想先知道：哪些会成功、哪些会失败、哪些疑似重复

**v4 方案**
- 本地先把原始文本解析成 `stagingItems`
- 本地显示列集合与样例记录
- 云端支持 `previewOnly=true`
- 统一返回 `valid / invalid / deduplicated / warnings / errors`
- 错误列表按行号展示，便于回源文件修复

**后续商业化增强**
- 生成可下载错误报告
- 支持只重试失败行
- 支持大文件异步任务

---

### 场景 G：标题写法不一致，容易导入重复题

**问题**
- 同一题可能只是空格、括号、标点、标题前缀不同
- 人工查重很难

**v4 方案**
- 对标题做归一化：移除空格、括号、连接符等
- 默认按归一化标题做去重
- 支持 `dedupeStrategy=skip|update`
- 若更新已有题，会自动提升 `version`

**后续商业化增强**
- 标题 + 题干双字段相似度查重
- 增加人工确认“可能重复”工作台

---

### 场景 H：后台管理体验需要更像产品，不只是 CRUD

**问题**
- 商业化前，运营需要信心：知道什么能搜、什么已归档、哪些还在待审核

**v4 方案**
- 状态拆分为 `status + reviewStatus + lifecycleState`
- 编辑页支持摘要、标题变体、OCR 文本、关联题、审核状态
- 列表页展示版本号、审核态、生命周期
- 归档时保留 `previousStatus / previousReviewStatus`，恢复时回到归档前状态
- `saveQuestion` / `importQuestions` 都会维护 `statusHistory`

**后续商业化增强**
- 角色权限与审批流
- 审核意见与驳回原因
- 版本 diff 和回滚
- 操作日志 / 消息通知 / 责任人工作台

## 3. 当前版本已落地的关键价值

- 搜索页不再只是“输入框 + 列表”，而是带分页、facets、分组和摘要的正式接口风格
- 导入不再只是“粘贴 JSON 写库”，而是加入了模板选择、本地暂存、字段映射、预检、去重和导入结果
- 管理端不再止于发布/删除，而是补上审核态、生命周期和版本化字段
- 文档层面对“为什么这样设计、未来如何落地”已经补齐

## 4. 距离真正商业化还缺什么

1. 真实 OCR / 图片上传链路
2. 索引化检索与大规模分页
3. Excel / CSV / Word 实际文件导入
4. 更可视化的字段映射与批量修复 UI
5. 审核流、角色权限、操作日志、版本 diff
6. 埋点与数据分析，验证哪些筛选/推荐真的有效
