# 上传后首日检查清单

这份清单面向 **“已经成功上传到微信开发者工具，但想把项目尽快跑顺、可演示、可继续上传新版本”** 的场景。

## 1. 先确认云环境

检查两处：

- `miniprogram/app.js`
- `project.config.json`

都应该指向同一个真实云开发环境 ID。

## 2. 创建集合

至少准备这 4 个集合：

1. `questions`
2. `admins`
3. `import_tasks`
4. `audit_logs`

说明：

- `questions` 是题库主表
- `admins` 决定后台是否可进入
- `import_tasks` 让导入页 / 后台首页能展示最近任务
- `audit_logs` 让后台首页 / 任务中心能展示审计轨迹

## 3. 首次打开后台，先拿 OPENID

上传后的第一步建议不是直接改代码，而是：

1. 打开小程序首页
2. 进入 **后台管理**
3. 页面会直接显示当前账号的 `OPENID`
4. 点击“复制管理员 JSON”
5. 把这条记录写入 `admins` 集合

推荐管理员记录：

```json
{
  "openid": "your-openid",
  "name": "Primary Admin",
  "enabled": true,
  "role": "super_admin",
  "ownerTeam": "内容运营"
}
```

## 4. 用导入页一键准备演示题库

导入页现在提供两种首日快速入口：

- **一键填入 9 条演示 JSON**
- **切换成 workbook 演示任务**

推荐顺序：

1. 打开导入页
2. 点击一键填入演示数据
3. 看“暂存就绪度”是否完整
4. 先执行“预检”
5. 预检通过后再执行“导入”

这样 `questions / import_tasks / audit_logs` 三条链路会一起更完整。

## 5. 首次验证建议

### 用户侧

- 首页按钮跳转是否正常
- 搜索结果是否来自云端；若失败是否优雅回退到 mock
- 详情页是否可打开

### 管理侧

- 后台页是否识别为管理员
- 列表页是否能看到导入后的题目
- 导入页是否出现最近任务回执
- 后台首页 / 任务中心是否出现任务与审计日志

## 6. 准备下一次上传前，建议做的 3 件事

1. 在导入页完成一次真实预检并保留回执
2. 在后台页确认管理员角色、权限矩阵、最近任务都展示正常
3. 用首页的“首次打开建议”卡片再过一遍，确保别人接手时也能快速跑起来

## 7. 如果你只想快速演示

最省事的路线：

1. 首页进入搜索页演示用户体验
2. 后台页复制 OPENID 并初始化管理员
3. 导入页一键填入演示题库并预检导入
4. 回到后台页展示统计、任务、审计日志

这条路径比手工建题、手工解释字段更顺，也更像真实产品首日上线流程。
